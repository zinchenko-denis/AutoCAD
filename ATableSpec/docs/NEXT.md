## Архитектура (гибрид — не менять без причины)
Тонкий C#-плагин (.NET Framework 4.8, `net48`, NuGet AutoCAD.NET 24.0.0 = AutoCAD 2021) +
замороженный Python-движок `dxf_spec.exe` (PyInstaller). Обмен — JSON через временные файлы.
Поставка — одна папка `ATableSpec.bundle` в `%APPDATA%\Autodesk\ApplicationPlugins\`.
Жёсткое требование: **установка = одна папка, запуск = одна команда, без доустановки софта.**
Конструктор git/ветки не знает — ставит из релиза `latest`.

## Команды
- `ATSPEC` — быстрая ведомость (Commands.cs + QueryForm.cs).
- `ATSPECREPORT` — отчёт по формулам через окно (ReportCommand.cs + ReportBuilderForm.cs).
- `ATSPECUPDATE` — принудительный пересчёт отчётных таблиц (ReportReactor.cs).
- **Авто-пересчёт** — реактор: правка блока → таблица обновляется сама по завершении команды.

## Файлы
**C# (`src/AtSpecPlugin/`):**
- `Plugin.cs` — `IExtensionApplication`; `Initialize → ReportReactor.Attach()`, `Terminate → Detach()`.
- `Commands.cs` — команда `ATSPEC`.  `QueryForm.cs` — окно быстрой ведомости.
- `ReportCommand.cs` — `ATSPECREPORT`: блоки→окно→движок→`AcDbTable`; читает ATTRIB **и
  динамические свойства блока**; строит списки полей/слоёв для окна; пишет определение отчёта в таблицу.
- `ReportBuilderForm.cs` — окно «Заголовок | Выражение»; «Выражение» = редактируемый combo
  с вставками `Object.«…»` (ввод свободный); источник по слою; фильтр; группировка; сортировка.
- `ReportReactor.cs` — `StoreDef/ReadDef` (определение отчёта в Xrecord словаря расширения таблицы),
  `Recompute`, команда `ATSPECUPDATE`, **авто-триггер** (Attach/Hook по DocumentManager,
  дебаунс на `CommandEnded`, `_busy`-реентрантность, `LockDocument`).
- `AtSpecPlugin.csproj` — net48, ref AutoCAD.NET 24.0.0.

**Python-движок (`engine/`):**
- `atspec_report.py` — вычислитель выражений (`Object.«поле»`/`Object.Name`, арифметика,
  литералы `«»`/`""`, конкатенация `+`, `Count`, `Sum`, `Iff`, сравнения, `row`, скобки;
  **ячейка без ведущего `=` — литерал-текст**) + раннер отчёта
  (фильтр → разворот по объектам → группировка → **сортировка групп → нумерация** → агрегация
  → производные строки `templates[]`).
- `dxf_spec.py` — контракт `action="report"` зовёт `atspec_report.run_report`.
- `mapping.yaml` — маппинг полей.  `test_atspec_report.py` — юнит-тест (фикстуры В-13/В-32 +
  регресс на нумерацию/литерал/«не содержит»). Локально: `PYTHONUTF8=1 python3 test_atspec_report.py`.

**Прочее:**
- `bundle/ATableSpec.bundle/PackageContents.xml` — `LoadOnAutoCADStartup="True"`, 3 команды.
- `docs/HYBRID.md` — архитектура/контракт. `docs/make_manual.py` — генератор PDF (reportlab+DejaVu),
  мануал **ред.1.4**. `docs/ATableSpec_manual.pdf`, `docs/ATableSpec_install.pdf`.
- `.github/workflows/check.yml` — на ветках (≠main): компиляция DLL + тест движка, без релиза.
  `build.yml` — на push в `main` **ИЛИ** `workflow_dispatch`: PyInstaller + публикация релиза
  `latest` (ассет `ATableSpec.bundle.zip`). Триггер-пути: `src/**`, `engine/**`, `bundle/**`,
  `.github/workflows/**` (docs НЕ триггерят).

## Контракт движка
- payload: `{ "blocks":[{name,layer,attributes}], "action":"report", "report":<def> }`
- запуск: `dxf_spec.exe --json -c mapping.yaml --in IN.json --out-json OUT.json`
- ответ: `{ ok, report:{ title, header, rows } }`
- def: `{ templates:[{ filter:[{field,op,value}], columns:[expr...], group_by:int|null,
  sort_by:(colIdx,"asc"|"desc")|null }] }`
- операторы фильтра: `=  ≠  >  <  ≥  ≤  содержит  «не содержит»` (движок понимает и символы,
  и `!=`/`>=`/`contains`/`not_contains`).

## История по сессиям
**Сессия 1 — авто-триггер.** Авто-пересчёт: правка блока → таблица сама по завершении команды.
Хук `ObjectModified`/`Appended`/`Erased` + `CommandEnded`/`CommandCancelled`, дебаунс,
`_busy`-реентрантность, `LockDocument`. **Фикс:** ловить и `BlockReference`, и `AttributeReference`
(правка ДЛИНА/ИМЯ — это ATTRIB, иначе не триггерило). Манифест → load-on-startup. Мануал → ред.1.4.

**Сессия 2 — фидбэк конструктора (6 багов + 2 доп):**
1. №п/п при сортировке шёл не по порядку → движок: сортировка групп ДО нумерации.
2. Из «Поля для Object» убраны деталировочные `DOBL/DOBR/KLL/KLR/L/R/UGL/UGR` (и из фильтра —
   список один), добавлены `Ширина/Высота`.
3. Доборники не давали длину → читаем **динамические свойства блока** (длина доборника — параметр,
   не ATTRIB). ATTRIB в приоритете при совпадении имён.
4. Столбец «Примечание»/пустое выражение роняли движок → правило «ячейка без `=` — литерал».
5. Фильтр «не работал» → причина: неочевидная галка «Фильтр». Убрана; фильтр по полю+значению.
6. Добавлен оператор «не содержит». Логика: `=`Р1 строго; `содержит`Р → Р1/Р2/…; `не содержит`Р — обратное.
   Доп: переменные убраны и из фильтра; слой `_Опоры`→`RF-опоры` — **в чертеже** (список слоёв =
   реальные слои чертежа; в коде не подменить — иначе фильтр слал бы имя, которого нет на объектах).

**Сессия 3 — выпадающий список полей в «Выражение».** Столбец «Выражение» → редактируемый
`DataGridViewComboBoxColumn`: пункты = вставки `Object.«…»`, ввод свободный (DataError подавлен,
введённый текст добавляется в Items, чтобы сохранялся).

Все правки: движок проверен локально (юнит-тест), C# скомпилирован на CI (`check`+`build` зелёные),
бандл пересобран и опубликован в `latest`.

## Этапы (план конструктора) и текущая работа
Развитие разбито на 3 этапа:
- **Этап 1** — доработка текущих моментов + новый функционал (7 пунктов ТЗ + уточняющее видео).
- **Этап 2** — форма отчёта: несколько отчётов в одной таблице (напр. стойки+ригеля вместе).
  Движок это УЖЕ умеет (`templates[]` конкатенируются) — нужен только UI.
- **Этап 3** — внешний вид таблицы AutoCAD; выбор 3 стандартных отчётов
  (спецификация/раскрой/заполнения); «реверс» — загрузка готовой таблицы обратно в построитель
  для правки (фундамент есть: определение хранится в таблице, `ReadDef`).

## Ключевые факты данных
- **Проверено по DXF (ezdxf, оба файла):** `ДЛИНА` — заполненный ATTRIB у стоек/ригелей/опор
  (опоры: `ДЛИНА`=150, `АРТИКУЛ`=«01 07 06»). У **доборников атрибута длины НЕТ** — только
  `ПРОФ`+`ПРИВЯЗКА`; длина у них в динам. параметре (live-only) либо геометрии. У кронштейнов
  `ДЛИНА`/`АРТИКУЛ` пустые. Заполнения: размер — один ATTRIB `РАЗМЕР_ЗАП` вида «ШиринаХВысота»
  (разделитель — кириллическая Х U+0425); отдельных Ширина/Высота нет → движок разбивает строку.
- В32: ригеля/стойки/кронштейны/заполнения/створки/_Ограждения (доборников/опор нет).
  В-13: дополнительно RF-доборники (120) и _Опоры (24); длины доборников продублированы
  в `DOBL/DOBR` родительских профилей (фрагмент «.../<артикул>/<длина>»).
- Блоки анонимно-динамические (`*U…`), марка из `ИМЯ`; `Object.Name ≈ ПРОФ`. Имя берётся через
  `DynamicBlockTableRecord` (не `br.Name`).
- Слои: `RF-ригеля/RF-стойки/RF-кронштейны/RF-заполнения/RF-створки/RF-доборники/_Ограждения/_Опоры`.
  Атрибуты: `ИМЯ/ПРОФ/ДЛИНА/МАРКИРОВКА/РАЗМЕР_ЗАП`.
- Деталировочные `DOBL/DOBR/KLL/KLR/L/R/UGL/UGR` — в спецификации не участвуют, скрыты из UI
  (но всё ещё читаются в records).
- Производные (В-13): крышка ригеля `Длина−51`, прижим `Длина−54`, раскрой `Длина−20`;
  крышка стойки `Длина−150`, литерал-артикул «01 02 04». **Константы — ДАННЫЕ проекта, плавают**
  (В-32: профили 17_07_04, длина 2748).
- СПДС-таблицы (SPDSTABLE/SPDSTABLE2): читается только ТЕКСТ ячеек; формулы/структура/запись — нет.
