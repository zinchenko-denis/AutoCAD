# ATableSpec — состояние и следующий шаг (хэндофф для новой сессии)

**Прочитать первым.** Рабочая записка для продолжения с чистым контекстом.

## TL;DR
- **Что:** плагин AutoCAD, строит ведомости/спецификации из блоков **без СПДС**.
- **Где:** публичный репо `github.com/zinchenko-denis/AutoCAD`, инструмент в `ATableSpec/`.
- **В `main` и в релизе `latest` сейчас:** движок (ядро выражений + `action=report`),
  команда `ATSPECREPORT` (свой отчёт по формулам), команда `ATSPECUPDATE` (пересчёт) +
  хранение определения отчёта в самой таблице, манифест регистрирует 3 команды, PDF-мануал.
- **Следующий шаг: авто-триггер реактора** (правка блока → таблица сама пересчиталась).
  Дизайн ниже готов — переизобретать не надо.

## Архитектура (гибрид — не менять без причины)
Тонкий C#-плагин (.NET Framework 4.8, `net48`, NuGet AutoCAD.NET 24.0.0 = AutoCAD 2021) +
замороженный Python-движок `dxf_spec.exe` (PyInstaller, встроенный Python). Обмен — JSON
через временные файлы. Поставка — одна папка `ATableSpec.bundle` в
`%APPDATA%\Autodesk\ApplicationPlugins\`. Жёсткое требование конструктора:
**установка = одна папка, запуск = одна команда, без доустановки софта.**

## Что уже сделано (в `main`, компилируется — CI check зелёный)
- `engine/atspec_report.py` — чистый Python (без ezdxf) вычислитель выражений
  (`Object.«поле»`/`Object.Name`, арифметика, литералы «»/"", конкатенация `+`,
  `Count`, `Sum`, `Iff`, сравнения, `row`, скобки) + раннер отчёта
  (фильтр → разворот по объектам → группировка → агрегация → сортировка) +
  производные строки (`templates[]`).
- `engine/dxf_spec.py` — контракт `action="report"` зовёт `atspec_report.run_report`.
- `src/AtSpecPlugin/ReportCommand.cs` — команда `ATSPECREPORT`: блоки → окно → движок →
  `AcDbTable`; при отрисовке сохраняет определение отчёта в таблицу (`ReportReactor.StoreDef`).
- `src/AtSpecPlugin/ReportBuilderForm.cs` — окно «Заголовок|Выражение» + источник по слою +
  группировка + сортировка.
- `src/AtSpecPlugin/ReportReactor.cs` — **костяк пересчёта** (авто-триггера ПОКА НЕТ):
  `StoreDef`/`ReadDef` (определение в Xrecord словаря расширения таблицы, чанки по 250) +
  `Recompute(doc, only)` (перечитать все блоки модели → для каждой таблицы с нашим
  определением вызвать движок `action=report` → перезалить ячейки `SetSize`+`GenerateLayout`) +
  команда `ATSPECUPDATE` (выбрать таблицы или все → пересчитать).
- `src/AtSpecPlugin/Plugin.cs` — `IExtensionApplication`, только сообщение о загрузке.
- `bundle/.../PackageContents.xml` — `LoadOnCommandInvocation="True"`, в `<Commands>` —
  ATSPEC + ATSPECREPORT + ATSPECUPDATE.
- `docs/ATableSpec_manual.pdf` — руководство; `docs/HYBRID.md` — архитектура/контракт.

В живом AutoCAD ещё не кликалось (раннер без CAD). Реальная валидация — ручной тест (ниже).

## СЛЕДУЮЩИЙ ШАГ — авто-триггер (дизайн готов)
Кладётся **поверх** `ReportReactor.Recompute`. Порядок: сперва убедиться, что `ATSPECUPDATE`
пересчитывает вживую (тест-протокол) — **только потом** автоматизировать триггер.

1. `PackageContents.xml`: `LoadOnAutoCADStartup="True"` (реактор подключается при старте;
   `LoadOnCommandInvocation` можно убрать). Компромисс: DLL грузится в каждой сессии →
   `Initialize` дёшев, реактор бездействует, пока нет таблицы с нашим определением.
2. `Plugin.cs`: `Initialize` → `ReportReactor.Attach()`; `Terminate` → detach.
3. `ReportReactor`:
   - `Attach()`: подписка на `DocumentManager.DocumentCreated`/`DocumentActivated` → `Hook(doc)`;
     + все текущие документы.
   - `Hook(doc)`: подписка на `doc.Database.ObjectModified` + `ObjectErased` + `ObjectAppended`
     и `doc.CommandEnded`; снятие на `BeginDocumentClose`.
   - обработчик изменения: если `_busy` → выход; если `e.DBObject is BlockReference` → `_dirty=true`.
   - `CommandEnded`: если `_dirty && !_busy` → `_dirty=false`; `_busy=true`;
     `using (doc.LockDocument()) { Recompute(doc, null); }`; `_busy=false`; всё в try/catch.
   - **Ре-энтрантность:** `_busy` гасит события от нашей же перезаписи таблицы.
     **Дебаунс:** пересчёт раз на завершение команды, не на каждое событие.
   - `LockDocument` обязателен — правка вне команды иначе кинет `eLockViolation`.
     В команде `ATSPECUPDATE` документ уже залочен, доп. лок не нужен.

## Тест-протокол (живой AutoCAD; обязателен перед «зелёным» на авто)
1. Поставить бандл из релиза `latest`. `ATSPECREPORT` → блоки → столбцы → таблица легла.
2. Изменить атрибут/длину блока → `ATSPECUPDATE` → значения обновились.
3. Скопировать таблицу в новый чертёж с такими же блоками → `ATSPECUPDATE` → переподтянулось.
4. (после авто-триггера) Изменить блок → таблица пересчиталась сама по завершении команды,
   без `ATSPECUPDATE`. Проверить отсутствие зацикливания/тормозов.

## Дальше после авто-триггера

## Ключевые факты модели/данных
- `ДЛИНА` — это ATTRIB («размер по ручкам блока»), читается напрямую (`Object.«Длина»`).
- Блоки анонимно-динамические (`*U…`), марка из `ИМЯ`; `Object.Name` ≈ `ПРОФ`.
- Проверенные производные (В-13): крышка ригеля `Длина−51`, прижим `Длина−54`,
  раскрой `Длина−20`; крышка стойки `Длина−150`, литерал-артикул «01 02 04».
  **Константы — это ДАННЫЕ проекта, плавают** (В-32: профили 17_07_04, длина 2748).
- СПДС-таблицы (SPDSTABLE/SPDSTABLE2): читается только ТЕКСТ ячеек; формулы/структура и
  запись — нет → их таблицы не зеркалим, конструктор задаёт формулы один раз, инструмент регенерит.
- Контракт движка: payload `{ "blocks":[{name,layer,attributes}], "action":"report", "report":<def> }`;
  запуск `dxf_spec.exe --json -c mapping.yaml --in IN.json --out-json OUT.json`;
  ответ `{ ok, report:{ title, header, rows } }`.

## Рабочий процесс / грабли
- Репо публичный → **клон без токена**. Токен (classic, repo+workflow) нужен только для push.
  **Токен в репозитории не хранится** — попросить свежий у Дениса в начале сессии
  (если push даёт 403 — токен отозван, нужен новый).
- Первый шаг сессии: `git clone`, прочитать этот файл + `docs/HYBRID.md`.
- Feature-ветки пушим сразу; **в `main` — только по явному «Пушим»** (мёрж триггерит релиз).
- Коммиты **unsigned**: `git config commit.gpgsign false` (signing даёт HTTP 400; `--gpg-sign` не пытаться).
- CI: `check.yml` на ветках (≠main) — сборка DLL + тест движка, без релиза.
  `build-bundle` на `main` — PyInstaller + публикация релиза `latest` (ассет `ATableSpec.bundle.zip`).
  Пути-триггеры: `ATableSpec/src/**`, `ATableSpec/engine/**`, `ATableSpec/bundle/**`,
  `.github/workflows/**` (docs НЕ триггерят).
- **Логи CI с раннера не вытащить** (хост не в allowlist). Падение компиляции — диагностировать
  чтением кода; падение движка — воспроизвести локально (Windows cp1252: тест делает
  `sys.stdout.reconfigure(utf-8)`, CI ставит `PYTHONUTF8=1`).
- Проверка CI — опрос `api.github.com/repos/zinchenko-denis/AutoCAD/actions/runs?branch=…`
  (токен из remote).
