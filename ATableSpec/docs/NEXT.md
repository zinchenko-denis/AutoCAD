## Архитектура (гибрид — не менять без причины)
Тонкий C#-плагин (.NET Framework 4.8, `net48`, NuGet AutoCAD.NET 24.0.0 = AutoCAD 2021) +
замороженный Python-движок `dxf_spec.exe` (PyInstaller). Обмен — JSON через временные файлы.
Поставка — одна папка `ATableSpec.bundle` в `%APPDATA%\Autodesk\ApplicationPlugins\`.
Жёсткое требование: **установка = одна папка, запуск = одна команда, без доустановки софта.**
Конструктор git/ветки не знает — ставит из релиза `latest`.

## Команды
- `ATSPEC` — быстрая ведомость (Commands.cs + QueryForm.cs).
- `ATSPECREPORT` — отчёт по формулам через окно (ReportCommand.cs + ReportBuilderForm.cs).
- `ATSPECEDIT` — **правка готовой таблицы**: выбрать таблицу → `ReadDef` → построитель ЗАПОЛНЕН (реверс
  `def`→форма, `FromDef`/`SeedFromSection`) → правка → по OK `StoreDef`+`Recompute([id])` пересобирает НА МЕСТЕ.
- `ATSPECEXPORT` — **выгрузка таблицы в CSV** (для внешнего оптимизатора раскроя Алексея): сырой дамп ячеек,
  разделитель `;`, UTF-8 с BOM, путь через диалог сохранения.
- `ATSPECUPDATE` — принудительный пересчёт отчётных таблиц (ReportReactor.cs).
- **Авто-пересчёт** — реактор: правка блока → таблица обновляется сама по завершении команды.

## Контракт движка
- payload: `{ "blocks":[{name,layer,attributes}], "action":"report", "report":<def> }`
- запуск: `dxf_spec.exe --json -c mapping.yaml --in IN.json --out-json OUT.json`
- ответ (Фаза 2): `{ ok, report:{ title, sections:[{ title, header, hide_header,
  header_merges, rows }] } }`. Для совместимости движок отдаёт ещё `header`/`rows` плоско
  (конкатенация секций) — старые потребители/тесты не сломаны.
- def (Фаза 2): `{ title, hide_title, scale, sections:[{ section_title, hide_header,
  header:[...], header_merges:[[s,e],...], columns:[expr...], filter:[{field,op,value}],
  group_by:int|null, sort_by:(colIdx,"asc"|"desc")|null }] }`.
  Источник секции (слой) кладётся в её `filter` как `Слой = X`. `=row` нумеруется ВНУТРИ
  секции. Старый формат `{header, templates:[...]}` движок принимает (мигрирует в одну секцию
  на шаблон) — уже вставленные таблицы пересчитываются без изменений раскладки.
- операторы фильтра: `=  ≠  >  <  ≥  ≤  содержит  «не содержит»` (движок понимает и символы,
  и `!=`/`>=`/`contains`/`not_contains`).

## История по сессиям
**Сессия 1 — авто-триггер.** Авто-пересчёт: правка блока → таблица сама по завершении команды.
Хук `ObjectModified`/`Appended`/`Erased` + `CommandEnded`/`CommandCancelled`, дебаунс,
`_busy`-реентрантность, `LockDocument`. **Фикс:** ловить и `BlockReference`, и `AttributeReference`
(правка ДЛИНА/ИМЯ — это ATTRIB, иначе не триггерило). Манифест → load-on-startup. Мануал → ред.1.4.

**Сессия 2 — фидбэк конструктора (6 багов + 2 доп):**
1. №п/п при сортировке шёл не по порядку → движок: сортировка групп ДО нумерации.
2. Из «Поля для Object» убраны деталировочные `DOBL/DOBR/KLL/KLR/L/R/UGL/UGR` (и из фильтра —
   список один), добавлены `Ширина/Высота`.
3. Доборники не давали длину → читаем **динамические свойства блока** (длина доборника — параметр,
   не ATTRIB). ATTRIB в приоритете при совпадении имён.
4. Столбец «Примечание»/пустое выражение роняли движок → правило «ячейка без `=` — литерал».
5. Фильтр «не работал» → причина: неочевидная галка «Фильтр». Убрана; фильтр по полю+значению.
6. Добавлен оператор «не содержит». Логика: `=`Р1 строго; `содержит`Р → Р1/Р2/…; `не содержит`Р — обратное.
   Доп: переменные убраны и из фильтра; слой `_Опоры`→`RF-опоры` — **в чертеже** (список слоёв =
   реальные слои чертежа; в коде не подменить — иначе фильтр слал бы имя, которого нет на объектах).

**Сессия 3 — выпадающий список полей в «Выражение».** Столбец «Выражение» → редактируемый
`DataGridViewComboBoxColumn`: пункты = вставки `Object.«…»`, ввод свободный (DataError подавлен,
введённый текст добавляется в Items, чтобы сохранялся).

Все правки: движок проверен локально (юнит-тест), C# скомпилирован на CI (`check`+`build` зелёные),
бандл пересобран и опубликован в `latest`.
